I'm Chris Davie - a data and software engineer, have designed algorithms, analysed data

Hope you're finding my portfolio interesting - would be good to hear from you (cjdavie@googlemail.com). Otherwise, cat pictures!1

(My linkedin, if you're here for that).

I've been a data engineer and software engineer for the last few years. Prior to that I was an algorithm design and data analysis freelancer and consultant, I started after finishing my PhD in computational physics at Imperial College London.

I hope my portfolio shows I enjoy solving problems, building useful products and discovering patterns and behaviour in complex systems. It's great when the things I've build get used - it's getting good, useful solutions out that I find most rewarding.

I also hope my portfolio shows an ongoing tendency to build solutions to pretty complicated problems and see them (often!) through to completion.

My preference for building working solutions means I aim for the more efficient, well established solutions. I've built novel systems - algorithm development and complex parallelisation during freelancing and my PhD, when I was doing something new and off-the-shelf solutions weren't available, or it needed to be really fast, but if there's a library or framework out there I'll try and use it.

I also prefer high quality code and test driven development, it takes more time up front but it's way easier to fix. I always try to deliver cost effective solutions for the people I work for, delivering the simplest working solution I can.

I find using these statistical and machine learning techniques really fascinating when applied to human behaviour - understanding what groups of people do, when and why, and how adjusting that can change behaviours. One of the most interesting papers I've read involves algorithmically predicting riots and civil unrest. It's amazing.

I could go on, but that's probably enough. Thanks for stopping by!
